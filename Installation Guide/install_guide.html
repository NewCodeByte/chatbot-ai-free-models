Quick Installation

  1. Upload Files

    Unzip the archive and upload the entire chatbot folder (e.g., chatbot-x/) to your web server in a location accessible via URL.

  2. Folder and File Permissions

    Ensure the web server has write permissions for:

    • The main chatbot folder (to create and write to the db.db database file).
    • The images/ subfolder (for uploading icons/logos).
    • The sounds/ subfolder (for uploading the notification sound).

    Verify that the db.db file, once created, is also writable by the web server process.

  3. First Access and Admin Setup

    Open your browser and navigate to the login page: http://yoursite.com/path_to_chatbot/login.php.

    On first access, the system will detect that no administrator exists and will present you with a registration form.

    Enter an Admin Username and a secure Password (and confirm it) to create your account. This account will be saved in the db.db database with the password protected by hashing.

    After creating the account, you will be redirected (or you can navigate) to the normal login page.

  4. Accessing the Admin Panel

    Once the account is created, log in at http://yoursite.com/path_to_chatbot/login.php using the credentials you just defined.

    After logging in, you will be redirected to the Control Panel (admin.php).

  5. Initial Configuration

    Inside the Control Panel:

    • Go to the Settings tab.
    • Enter your API Key from OpenRouter.ai.
    • Select an AI Model (many free models are available).
    • Customize the Knowledge Base and Behavior Rules. Consult the Guide Tab in the panel for templates and suggestions.
    • Explore the other tabs (Style, Images, Sounds) for further customization.
    • Select the desired Interface Language.

  6. Integrate into Your Site

    Copy the script line provided in the Guide Tab of the admin panel. It should look similar to:

    <script src="/path_to_chatbot_folder/embed.js" defer></script>

    Paste this line into the pages of your website where you want the chatbot to appear, preferably before the closing </body> tag. Ensure the path to the embed.js script is correct.

File Structure (Main)

  • admin.php: The administration panel.
  • chatbot.php: Handles the chatbot logic and interface.
  • embed.js: Script for website integration.
  • style.css: Stylesheets.
  • admin.js & chatbot.js: JavaScript files.
  • login.php: Login and initial setup page.
  • export.php: Message export.
  • db_init.php: Database initialization script.
  • db.db: SQLite database (created on first run).
  • /images/: Images folder.
  • /sounds/: Sounds folder.
  • /languages/: Translations folder.
  • install_guide.html: This guide file.

Security

  • Password Hashing: Passwords are now stored securely using hashing.
  • Directory Protection: Always consider protecting specific files like admin.php with additional methods (e.g., .htaccess on Apache servers) if you require a higher level of security, such as limiting access to specific IP addresses or requiring additional HTTP authentication. Direct database access should already be protected by the provided .htaccess file.
  • File Permissions: Set permissions to the minimum necessary for operation, ensuring write access only where needed (main folder for db.db, images/, sounds/).

Troubleshooting

Consult the "Troubleshooting Common Issues" section in the Guide Tab of the administration panel.

  • DB Error on first access to login.php: Ensure the plugin folder is writable by the web server to allow the creation of the db.db file.
  • Other issues: check your server's PHP error logs.
